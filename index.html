Why we built this

We loved Omarchy — the keybindings, the workflow, the idea of a curated Arch desktop. But we wanted something leaner. Fewer packages, lower memory footprint, support for more than one compositor. We tried to contribute those changes upstream, but the project has a different vision: it's a personal setup built for one person's preferences, and that's totally fair. So we forked, rewrote most of the stack, and built smplOS.

What started as a trim-down turned into something new. We wrote a suite of native GUI apps in Rust — a start menu, notification center, app center, display manager, keyboard manager — each under 5 MB, starting in milliseconds. No Electron, no GTK monoliths. Tiling WM users have gone without these tools for years because the only options were heavy desktop-environment panels. Now they're lightweight, theme-aware, and work on both X11 and Wayland.

We built a modular edition system — productivity, creators, communication, development, AI — that stacks cleanly on top of a minimal base. Adding a new edition is just a package list and a post-install script, not a fork of the entire OS. The base idles under 800 MB of RAM, but you can stack every edition and still have a lighter system than most mainstream distros.

You can spin up sandboxed web apps on the fly — any website becomes an isolated app with its own window, its own profile, and no access to your main browser data. There's a 14-theme system that recolors every app on the desktop with one command. A suckless terminal patched for inline images at a fraction of Kitty's memory. Offline-first installation — no internet, under 2 minutes. Cross-compositor architecture so the same configs, themes, and keybindings work across Hyprland (Wayland) and DWM (X11).

Same muscle memory as Omarchy. Different philosophy. Minimal by design, not by accident.

Why smplOS?



Lightweight

Under 800 MB of RAM on a cold boot with the full desktop running — bar, notifications, compositor, and all background services. Light enough for a 2 GB VM, comfortable enough for daily driving.



Offline-first

The ISO carries every package. No internet needed during install. A fresh install completes in under 2 minutes on any hardware. Plug in, boot, done.



Cross-compositor

Built from the ground up for multiple compositors. Hyprland (Wayland) ships first, DWM (X11) is next. Shared configs, shared themes, shared keybindings — the compositor is just a thin layer.



One UI toolkit

EWW powers the bar, widgets, and dialogs. It runs on both X11 and Wayland. No waybar, no polybar, no redundant tools. One codebase for everything.



14 themes

One command switches colors across the entire system — terminal, bar, notifications, borders, lock screen, btop, neovim, and VS Code. Every theme includes matching wallpapers.



Sandboxed web apps

Turn any website into an isolated app with its own window, profile, and sandbox — no access to your main browser data. Create and manage them from the built-in Web App Center. Chat apps, dashboards, anything.

Built-in Rust Apps

Tiling WM users have gone without graphical settings panels for years. The only options were Electron wrappers or full GNOME/KDE panels. So we wrote our own — native Rust + Slint, each under 5 MB, starting in milliseconds.

Start Menu

Browse apps by category, search across all installed packages (AUR, Flatpak, AppImage, web apps), and access system settings. Full keyboard navigation. Theme-aware colors and border radius.

App Center

Search and install apps from Flathub with a single click. Browse by category, see descriptions and download sizes, install or remove Flatpak apps without touching the terminal.

Web App Center

Turn any website into a sandboxed app with its own window and browser profile. Manage VPN routing, secure mode, and delete apps you no longer need. All from one GUI.

Notification Center

Groups desktop notifications by app with dismiss, clear-all, and scrollable history. Integrates with Dunst over D-Bus. Double-click to launch the source app. Idles at under 10 MB of RAM.

Display Manager

Detects connected monitors, shows resolution, refresh rate, scale, and position. Apply changes live — no CLI, no config file editing. Just hyprctl under the hood.

Keyboard Manager

Add and remove keyboard layouts, toggle NumLock, adjust repeat rate and delay — all applied live. Previously this required hand-editing config files and reloading the compositor.
